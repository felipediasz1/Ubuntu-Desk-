Autenticação

Todos os endpoints abaixo exigem autenticação. Dois métodos suportados:

MétodoHeaderValidade
Bearer tokenAuthorization: Bearer <token>30 dias — via POST /api/login
API Key nomeadaX-Api-Key: <key>Permanente — gerenciado abaixo
# Obter Bearer token
curl -X POST {{ base_url }}/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"SUA-SENHA"}'

# Usar em requisições
curl {{ base_url }}/api/peers \
  -H "Authorization: Bearer TOKEN" \
  # ou
  -H "X-Api-Key: ud_..."

Endpoints — Dispositivos

GET /api/peers Listar todos os dispositivos 
curl {{ base_url }}/api/peers \
  -H "X-Api-Key: ud_..."

# Resposta
{
  "total": 5,
  "peers": [
    { "id": "123456789", "hostname": "PC-ESCRITORIO", "os": "Windows 11",
      "username": "joao", "online": true, "note": "", "created_at": "..." }
  ]
}
GET /api/peers/<id> Detalhes de um dispositivo 
curl {{ base_url }}/api/peers/123456789 \
  -H "X-Api-Key: ud_..."
PUT /api/peers/<id> Atualizar nota do dispositivo 
curl -X PUT {{ base_url }}/api/peers/123456789 \
  -H "X-Api-Key: ud_..." \
  -H "Content-Type: application/json" \
  -d '{"note":"Sala de reunião 3"}'
DELETE /api/peers/<id> Remover dispositivo do servidor 
curl -X DELETE {{ base_url }}/api/peers/123456789 \
  -H "X-Api-Key: ud_..."

Endpoints — Stats & Auditoria

GET /api/stats Resumo do sistema 
curl {{ base_url }}/api/stats -H "X-Api-Key: ud_..."

# Resposta

  "peers":      { "total": 10, "online": 3, "offline": 7 },
  "audit":      { "events_today": 24, "login_failures": 1 },
  "recordings": { "count": 5 }

GET /api/audit Listar eventos de auditoria
ParâmetroTipoDescrição
categorystringopcionalsecurity | access | admin | system
searchstringopcionalBusca livre em action/detail
limitintopcionalMáx registros (padrão: 100, max: 500)
offsetintopcionalPaginação
curl "{{ base_url }}/api/audit?category=security&limit=50" \
  -H "X-Api-Key: ud_..."
